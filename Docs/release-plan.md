# Release Plan — Lumina University Management System

## Огляд релізів

| Реліз  | Версія | Дата       | Фокус                                          |
|--------|--------|------------|------------------------------------------------|
| MVP    | v0.1   | Місяць 1   | Автентифікація, студенти, базова навігація     |
| Alpha  | v0.5   | Місяць 2   | Курси, розклад, оцінки, відвідуваність         |
| Beta   | v0.8   | Місяць 3   | Звіти, сповіщення, рольові обмеження           |
| Release| v1.0   | Місяць 4   | Стабілізація, тести, документація, production  |

---

## v0.1 — MVP

**Мета:** Базова робоча система з автентифікацією та управлінням студентами.

Функції:
- ASP.NET Core Identity: реєстрація, вхід, вихід
- Ролі: Admin, Teacher, Student
- CRUD студентів (тільки Admin)
- CRUD факультетів та груп
- SQLite за замовчуванням, підтримка SQL Server
- Автоматичний seed тестових користувачів
- Глобальний ExceptionMiddleware

---

## v0.5 — Alpha

**Мета:** Повноцінний навчальний процес — курси, розклад, оцінки.

Функції:
- CRUD курсів з прив'язкою до викладача та груп
- Типи занять: Lecture, Seminar, Laboratory (Factory Pattern)
- Розклад (Schedule) з прив'язкою до групи та заняття
- Система оцінювання (Grade) — виставлення та перегляд
- Відвідуваність (Attendance) — відмітка та статистика
- Стратегії рейтингу: Exam, Attendance, Scholarship (Strategy Pattern)
- Сторінка деталей студента з повною інформацією

---

## v0.8 — Beta

**Мета:** Звітність, сповіщення, полірування UX.

Функції:
- Генерація звітів PDF та Excel (Builder Pattern)
- Список звітів з історією
- SignalR сповіщення в реальному часі
- Observer Pattern для розсилки сповіщень
- Composite для ієрархічного відображення факультет → група → студент
- Visitor для обходу ієрархії (рейтинг, відвідуваність, статистика)
- Dashboard з агрегованою статистикою
- Пошук по студентах та курсах

---

## v1.0 — Production Release

**Мета:** Стабільна, задокументована, протестована система.

Функції:
- 26+ unit-тестів (xUnit, FluentAssertions, Moq)
- Покриття: сервіси, фабрики, контролери
- Повна документація (README, DEVELOPER_GUIDE, USER_GUIDE, FINAL_REPORT)
- Діаграми класів та sequence діаграми
- Підтримка SQLite та SQL Server через `appsettings.json`
- Обробка статус-кодів (404, 500) через middleware
- Валідація даних на рівні моделей та сервісів

---

## Критерії готовності (Definition of Done)

- Функціонал реалізований і протестований
- Unit-тести написані і проходять
- Код пройшов code review
- Документація оновлена
- Build проходить без помилок і попереджень
